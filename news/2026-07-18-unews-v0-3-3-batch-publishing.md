uNews больше не растягивает готовую очередь на несколько четырёхчасовых интервалов.

Что изменилось:
— один запуск публикует до 20 готовых патчноутов;
— FIFO между проектами и порядок версий внутри проекта сохраняются;
— между Telegram-постами выдерживается не меньше 61 секунды;
— после каждого принятого Telegram-поста published.json сразу коммитится в GitHub;
— если checkpoint состояния не прошёл, пакет останавливается до следующего поста;
— ошибка ранней версии по-прежнему блокирует только её проект.

Что это даёт:
— очередь из 20 готовых патчей выходит примерно за 20 минут;
— GitHub по-прежнему просыпается планово только раз в четыре часа;
— посты визуально разнесены по разным минутам;
— сбой после одного поста не должен приводить к продолжению по несохранённому состоянию.

Короткий текст для Telegram:

Патч uNews 0.3.3: готовая очередь публикуется пакетом до 20 постов, но с паузой не меньше 61 секунды и обязательной фиксацией каждого результата перед продолжением.
