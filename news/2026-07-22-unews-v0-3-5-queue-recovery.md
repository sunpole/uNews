# uNews 0.3.5: очередь больше не теряет причину сбоя

Исправлен операционный дефект GitHub Actions, из-за которого реальный запуск мог завершиться до фиксации `data/health.json` и `data/errors.json`. Служебный Issue сообщал об ошибке, но полезное состояние оставалось устаревшим.

Перед исправлением проведена безопасная диагностика:

- `npm test` прошёл без ошибок;
- dry-run проверил 35 публичных репозиториев;
- обнаружено 8 готовых uDream-патчноутов;
- ошибок очереди, policy и изображений не найдено;
- Telegram `getMe` подтвердил бота `@uNewsDev_bot`;
- Telegram `getChat` подтвердил канал `@uNewsLog`;
- токены и значения secrets в отчёты не выводились.

Что изменено:

- `publish:all` запускается через отдельный recovery-runner;
- при фатальном сбое безопасная причина сохраняется в `health.json` и `errors.json`;
- token-like строки автоматически редактируются;
- неуспешная попытка не подменяет время последней успешной проверки;
- Telegram preflight больше не обрывает workflow до записи состояния;
- workflow сначала коммитит изменения `data/`, а только затем возвращает ошибку;
- успешный запуск очищает `last_error` и автоматически закрывает служебный Issue;
- тесты выполняются перед каждым запуском, включая расписание;
- добавлены регрессионные тесты recovery-state.

Не изменены:

- строгий FIFO-порядок;
- порядок версий внутри проекта;
- лимит до 20 публикаций за запуск;
- пауза не менее 61 секунды;
- немедленный GitHub checkpoint после каждого успешного Telegram-поста;
- запрет реальной локальной публикации;
- правила секретов и приватных данных.

После объединения патча будет выполнена реальная очередь. Восемь накопившихся uDream-патчноутов должны публиковаться последовательно от `23.8.0` до `23.8.7`, а каждый результат будет подтверждён `message_id`, `post_url` и GitHub checkpoint.

Короткий текст для Telegram:

Исправление uNews 0.3.5 восстанавливает надёжную публикационную очередь: при любом сбое причина теперь сохраняется в GitHub до завершения workflow, секреты удаляются из диагностики, а успешный запуск автоматически закрывает служебный Issue. Проверены 8 готовых uDream-патчноутов, Telegram-бот и канал; FIFO, пауза 61 секунда и checkpoint после каждого поста сохранены.
