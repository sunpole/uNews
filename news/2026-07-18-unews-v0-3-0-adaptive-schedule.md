uNews больше не обязан постоянно опрашивать GitHub четыре раза в час.

Что сделано:
— плановая проверка выполняется один раз в сутки;
— после реальной публикации очередь сама продолжает работу через десять минут;
— цепочка прекращается, когда готовых новостей больше нет;
— один проход по-прежнему публикует не больше одного поста;
— ручной запуск позволяет немедленно обработать срочную новость.

Короткий текст для Telegram:
Обновление uNews 0.3.0: при простое бот проверяет проекты раз в сутки, а после реальной публикации сам продолжает активную очередь с безопасной паузой. Срочную новость можно запустить вручную.
