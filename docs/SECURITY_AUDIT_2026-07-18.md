# Аудит безопасности uNews — 2026-07-18

## Итог

На момент проверки признаков публикации действующих секретов не обнаружено.

Проверены текущая ветка и все 128 коммитов, доступных во всех удалённых refs репозитория. Поиск выполнялся без вывода возможных значений: отчёт фиксировал только наличие совпадений и пути файлов.

## Проверено

- файлы локального окружения и распространённые контейнеры ключей;
- строки, похожие на Telegram bot tokens, GitHub tokens, API tokens, AWS и Google credentials;
- заголовки приватных ключей;
- обработка ошибок Telegram;
- GitHub Actions logs последней реальной публикации;
- права каждого workflow;
- передача credentials только нужным шагам;
- защита FIFO-патчноутов от случайного текста с credential-like значениями;
- зависимости GitHub Actions.

## Результаты

- совпадений с token-like значениями в доступной истории нет;
- чувствительные имена файлов в истории не обнаружены;
- Telegram credentials действительны: последняя безопасная диагностика получила успешные ответы bot identity и channel access;
- реальные значения credentials не печатаются;
- dry-run и pull-request workflow не получают Telegram credentials;
- publisher использует только `contents: write` и `issues: write`;
- Pages использует только `contents: read`, `pages: write` и `id-token: write`;
- все используемые GitHub Actions закреплены на проверенных commit SHA.

## Усиления

- расширен `.gitignore` для вариантов environment-файлов, private keys и credential JSON;
- policy блокирует дополнительные распространённые форматы credentials и URL с паролем;
- полный caption больше не выводится в Actions logs;
- непубличный channel identifier маскируется;
- добавлены `SECURITY.md` и `.github/CODEOWNERS`;
- добавлены тесты новых сигнатур.

## Ограничения проверки

Зашифрованные значения GitHub Secrets невозможно и не нужно читать обратно. Их наличие и актуальность подтверждаются только успешной авторизацией. Автоматическая текстовая проверка не распознаёт секрет, визуально нарисованный внутри изображения; поэтому screenshots перед публикацией должны оставаться без токенов, QR-кодов, паролей и приватных данных.

Для принудительного применения CODEOWNERS следует включить branch protection с обязательным review владельца для `main`.
